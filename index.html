Perfil y enfoque

La parte personal ya no vive separada del trabajo, sino explicando como se construye

El foco esta en IA aplicada, criterio tecnico, presentacion clara y una separacion limpia entre demo publica y producto mas serio.

Como trabajo

1. Problema antes que tecnologia

Primero se define el caso de uso, luego el nivel correcto: demo, producto, o componente de soporte.

2. Cierre tecnico visible

README, entrypoint, tests y narrativa publica deben decir lo mismo que el codigo.

3. Rigor con limites

Los dominios sensibles se presentan con cautela: legal, compliance, psicologico, educativo o regulatorio.

Siguiente paso

Si un caso encaja con automatizacion, RAG, analitica o agentes verticales, la web ya separa bien donde mirar primero: demos publicas o productos maduros.
